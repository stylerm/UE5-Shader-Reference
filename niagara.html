Scratch Pad Module HLSL
Scratch Pad Basics
✓Niagara uses a restricted HLSL subset — not the same as Material Custom nodes
✓Access particle attributes via Parameters.AttributeName — not raw variable names
✓ExecutionIndex gives the current particle index for per-particle logic
✓Map inputs/outputs explicitly in the Scratch Pad module UI
✓Use Emitter.LocalSpace / Emitter.Age for emitter-level data
✓Write results to output parameters — no return statement
Scratch Pad Access Pattern
// Reading particle attributes
float3 Pos = Parameters.Position;
float  Age = Parameters.Age;
float  Life = Parameters.Lifetime;
float  DT = Parameters.DeltaTime;
int    ID = Parameters.UniqueID;

// Writing output attributes
OutputParameters.Position = Pos + Velocity * DT;
OutputParameters.Color = float4(1,0,0,1);

// No return statement — write to outputs
Niagara Random Intrinsics

Niagara provides seeded random functions (not HLSL standard):

RandomUInt(seed)Seeded random integer — deterministic per particle
RandomFloat(seed)Seeded random float [0,1]
RandomUnitFloat3(seed)Random direction on unit sphere
Parameters.UniqueIDUse as hash seed for per-particle randomness
Restrictions & Things to Avoid
Niagara HLSL · Never Do This
✗No texture sampling in Particle Update / Spawn stages — GPU sim doesn't allow it
✗No dynamic loop bounds — use [unroll] or fixed iteration counts
✗No ddx / ddy — these are pixel shader only; Niagara sim runs in compute
✗No return statements in Scratch Pad — write to output parameters instead
✗No struct definitions in Scratch Pad — use flat parameter lists
✗Don't assume Material intrinsics — many UE material functions aren't available
✗No UAV writes in standard modules — data flows through typed SRV buffers only
Simulation vs Render Stage
!Simulation HLSL (Spawn/Update) runs as compute — no texture sampling, no derivatives
!Render HLSL (Particle Render) differs from simulation HLSL — different available functions
!Data Interface HLSL runs in its own context — check DI docs for available functions
✓Branching is allowed — use BRANCH hint for dynamic branching, FLATTEN to evaluate both
✓Switch statements work in Niagara HLSL — useful for loop mode selection
What IS Available
✓Standard math: lerp, saturate, smoothstep, clamp, abs, sign, pow
✓Vector math: dot, cross, normalize, length, distance
✓Trig: sin, cos, tan, atan2, asin, acos
✓Floor/ceil: floor, ceil, round, frac, fmod
✓Flow control: BRANCH, FLATTEN, UNROLL, LOOP hints
✓Typed buffer reads: Buffer<float> SRV access via index
Stateless Module Patterns · From Real Code
Stateless Particle Struct

Stateless modules use FStatelessParticle + FStatelessParticleContext — different from Scratch Pad:

// Stateless module entry point
void MyModule_Initialize(
    inout FStatelessParticle Particle,
    in FStatelessParticleContext Context)
{
  // Read state
  uint Idx = Context.ExecIndex;
  float3 Pos = Particle.Position;
  float Age = Particle.Age;

  // Write state directly
  Particle.Position = NewPos;
  Particle.PreviousPosition = OldPos;
  Particle.Scale = float3(1,1,1);
  Particle.Color = float4(1,0,0,1);
  Particle.MeshOrientation = NewQuat;
}
GPU Buffer Access Pattern

Stateless modules use typed Buffer<float> SRVs — read 3 or 4 consecutive floats per element:

// Reading float3 from typed float buffer
float3 ReadFloat3FromBuffer(
    Buffer<float> InBuffer, uint Index)
{
  uint Base = Index * 3;
  return float3(
    InBuffer[Base + 0],
    InBuffer[Base + 1],
    InBuffer[Base + 2]);
}

// Reading float4 (color, quaternion)
float4 ReadFloat4FromBuffer(
    Buffer<float> InBuffer, uint Index)
{
  uint Base = Index * 4;
  return float4(
    InBuffer[Base + 0], InBuffer[Base + 1],
    InBuffer[Base + 2], InBuffer[Base + 3]);
}
Indexing Strategies

Common patterns for selecting which array element a particle uses:

// Sequential (round-robin by particle ID)
uint Idx = ParticleID % NumPositions;

// Sequential with clamping (no wrap)
uint Idx = min(ParticleID, NumPositions - 1);

// Random via MurmurHash3 finalizer
uint Hash = ParticleID;
Hash ^= Hash >> 16;
Hash *= 0x85ebca6b;
Hash ^= Hash >> 13;
Hash *= 0xc2b2ae35;
Hash ^= Hash >> 16;
uint Idx = Hash % NumPositions;

// Age-based (interpolate through array)
float T = NormalizedAge * (float)Count;
uint Idx = min((uint)T, Count - 1);

Patterns & Tips
Useful Patterns
✓Random per-particle — use Parameters.UniqueID as seed with a hash function
✓Custom module — wrap HLSL in a Scratch Pad module for reuse across emitters
✓Quaternion math — multiply quaternions manually (no built-in qmul)
✓Jitter — pre-bake random sphere offsets: RandomUnitFloat3(seed) * Radius
✓Loop modes — implement Once/Loop/PingPong via switch on age-based index
✓Early exit — guard empty arrays: if (NumPositions == 0) return;
Quaternion Multiply (Helper)

No built-in quaternion multiply in Niagara HLSL — use this:

float4 QuatMultiply(float4 Q1, float4 Q2)
{
  return float4(
    Q1.w*Q2.x + Q1.x*Q2.w + Q1.y*Q2.z - Q1.z*Q2.y,
    Q1.w*Q2.y - Q1.x*Q2.z + Q1.y*Q2.w + Q1.z*Q2.x,
    Q1.w*Q2.z + Q1.x*Q2.y - Q1.y*Q2.x + Q1.z*Q2.w,
    Q1.w*Q2.w - Q1.x*Q2.x - Q1.y*Q2.y - Q1.z*Q2.z
  );
}

// Usage: combine rotations
Particle.MeshOrientation =
  QuatMultiply(BaseRot, DeltaRot);
Debugging Tips
!Debug via Color — write intermediate values to Particle.Color to visualize
!Scale debugging — set Particle.Scale = float3(value, value, value) to see ranges
!NaN check — if particles disappear, check for 0/0 or sqrt(negative)

✓Guard empty buffers — always check if (Count == 0) return; before buffer reads
